## 注意事项

### 版本号说明

给定版本号 MAJOR.MINOR.PATCH，递增规则如下：

1. **MAJOR** 版本：当你做了不兼容的 API 修改
2. **MINOR** 版本：当你做了向下兼容的功能性新增
3. **PATCH** 版本：当你做了向下兼容的问题修正

### 变更类型

- **新增** (Added) - 新功能
- **变更** (Changed) - 对现有功能的变更
- **弃用** (Deprecated) - 即将移除的功能
- **移除** (Removed) - 已移除的功能
- **修复** (Fixed) - Bug 修复
- **安全** (Security) - 安全相关的修复
